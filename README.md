# AISD_L3 - Интерактивная программа для решения задач по графам

## Описание

Консольная программа на Python для решения 4 заданий по алгоритмам и структурам данных (теория графов).

## Задания

### Задание 1: Поиск в глубину (DFS)
Файл: `tasks/task_1_dfs.py`

**Описание:**
- Применить алгоритм поиска в глубину к графу
- Записать время начала (pre-значения) и время конца (post-значения) обработки каждой вершины
- Пометить каждое ребро как:
  - **Древесное** - ребро в дереве поиска
  - **Прямое** - ребро к потомку
  - **Обратное** - ребро к предку
  - **Перекрёстное** - остальные ребра

**Ввод:** Граф в виде списка ребер (u v)

**Вывод:**
- Таблица с pre и post значениями для каждой вершины
- Классификация всех ребер

### Задание 2: Поиск в ширину (BFS)
Файл: `tasks/task_2_bfs.py`

**Описание:**
- Применить алгоритм поиска в ширину к графу
- Определить цикличность графа
- Исключить все дуги, не входящие в цикл

**Ввод:** Граф в виде списка ребер (u v)

**Вывод:**
- Расстояния и родительские вершины для BFS
- Определение цикличности (циклический/ациклический)
- Список ребер в циклах и ребер не в циклах

### Задание 3: Алгоритм Дейкстры
Файл: `tasks/task_3_dijkstra.py`

**Описание:**
- Найти кратчайшие пути из начальной вершины до всех остальных
- Использовать алгоритм Дейкстры

**Ввод:**
- Взвешенный граф (u v weight)
- Начальная вершина (по умолчанию 'f')

**Вывод:**
- Таблица с расстояниями и путями до каждой вершины

### Задание 4: Обход графа (дерева)
Файл: `tasks/task_4_traversal.py`

**Описание:**
- Записать последовательность вершин при различных видах обхода дерева:
  1. **Прямой (префиксный) обход** - родитель, затем потомки
  2. **Симметричный (внутренний) обход** - левое поддерево, родитель, правое поддерево
  3. **Обратный (постфиксный) обход** - потомки, затем родитель
  4. **Обход в ширину (BFS)**

**Ввод:** Дерево в виде списка ребер (parent child)

**Вывод:**
- Последовательности вершин при каждом типе обхода
- Визуализация структуры дерева

## Требования к вводу

### Формат ребер для заданий 1, 2, 4:
```
u v
```
где `u` и `v` - имена вершин (символы или числа)

Пример:
```
a b
a c
b c
```

### Формат ребер для задания 3:
```
u v weight
```
где `u` и `v` - имена вершин, `weight` - целое неотрицательное число (вес ребра)

Пример:
```
a b 5
a c 3
b c 2
```

### Завершение ввода:
Введите пустую строку, чтобы завершить ввод графа.

## Возможные ошибки и их решение

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `Ошибка: введите ровно две вершины (u v)` | Неверный формат ввода | Проверьте, что вы вводите ровно две вершины, разделённые пробелом |
| `Граф пуст! Невозможно выполнить поиск.` | Не введено ни одного ребра | Введите хотя бы одно ребро перед тем как завершить ввод |
| `Ошибка: вершина 'X' не существует в графе` | Указана неправильная стартовая вершина | Выберите вершину из доступного списка |
| `Ошибка: вес должен быть числом` | Вес ребра не является числом | Убедитесь, что вес задан целым числом |
